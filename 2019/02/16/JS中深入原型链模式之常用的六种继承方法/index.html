JS中深入原型链模式之常用的六种继承方法

发表于 2019-02-16 | 分类于 JS基础
字数统计: 856 | 阅读时长 ≈ 4

原型继承

原型继承是我们JS中最常用的一种继承方式
子类B想要继承父类A中的属性和方法（私有+ 公有），只需要让
B.prototype = new A ;即可。
原型的特点：它是把父类中私有的 + 公有的都继承到了子类原型上（子类公有的 ）


核心：原型继承并不是把父类中的属性和方法克隆一份一模一样的给B，而是让B和A之间增加了原型链的连接，以后B 的实例n想要A中的getX方法，需要一级一级的向上进行查找来使用。

父类：
function A() { this.x = 100; } A.prototype.getX = function(){ console.log(this.x); }
子类：
function B(){ this.y = 200; }
子类继承父类
B.prototype = new A;
最后修改Constructor的指向
B.prototype.constructor = B ;

call继承

把父类私有的属性和方法克隆一份一模一样的，作为子类私有的属性

冒充对象继承

把父类私有的 + 公有的（这里是可控制的，用hasOwnProperty()或propertyIsEnumberable()做筛选即可)，克隆一份一模一样的的给子类私有的。

混合模式继承

原型继承 + call继承
特点：

  1. 原型继承，可以继承父类私有的 + 公有的属性和方法；
  2. call继承：只能继承父类私有的属性和方法；
    缺点：
    父类私有的属性和方法即存在于子类的私有中，又存在于子类的公有中。

寄生组合式继承

call继承 + Object.create(proObject)
特点：

  1. call继承：只能继承父类私有的属性和方法；
  2. Object.create(proObject)：将父类的原型作为子类的原型，即继承父类公有的属性和方法
    缺点：
  3. A的原型改变，B的原型也会跟着改变，反之亦如此。

中间类继承法

将数组原型作为一个类数组实例的原型arguments.__proto__ = Array.prototype;，则arguments可以直接使用Array中的内置方法。
